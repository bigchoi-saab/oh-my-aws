### Learnings
1. Kiro Skills frontmatter is minimal (name + description required only). allowed-tools is experimental and bugged for workspace skills (#6055)
2. Tool access control lives in Agent JSON, not Skills. Must create .kiro/agents/ config with tools/allowedTools fields
3. MCP config at .kiro/settings/mcp.json (not .kiro/mcp.json as originally assumed)
4. AWS official MCP servers (awslabs.*) available via uvx - can be used immediately without custom MCP server
5. autoApprove field in MCP config enables 3-tier approval gate without custom MCP protocol extensions
